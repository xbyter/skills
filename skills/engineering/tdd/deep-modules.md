# 深模块

来自"A Philosophy of Software Design"：

**深模块** = 小接口 + 大量实现

```
┌─────────────────────┐
│   Small Interface   │  ← 少量方法，简单参数
├─────────────────────┤
│                     │
│                     │
│  Deep Implementation│  ← 复杂逻辑被隐藏
│                     │
│                     │
└─────────────────────┘
```

**浅模块** = 大接口 + 少量实现（避免）

```
┌─────────────────────────────────┐
│       Large Interface           │  ← 很多方法，复杂参数
├─────────────────────────────────┤
│  Thin Implementation            │  ← 只是传递
└─────────────────────────────────┘
```

设计接口时，问自己：

- 能否减少方法数量？
- 能否简化参数？
- 能否在内部隐藏更多复杂性？
